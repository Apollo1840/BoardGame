### 怪物数值设计技能说明

#### 技能名称
怪物数值设计（Monster Stat Allocation）

#### 输入参数
- **怪物条目**：包含以下字段
  - `level`：怪物等级（0-5）
  - `magic`：配置分数（浮点数，如1.5、0.5等）
  - `skills`：技能列表（JSON数组，每个技能有名称、效果等）
  - `attributes`：反应属性（JSON对象，可能为空）
- **表格**：Monster design table（已提供，包含各等级下不同magic对应的数值总数、攻击上限、防御上限，以及不同技能数量对应的效果品阶和配置）

#### 输出
- `attack`：分配后的攻击力
- `defence`：分配后的防御力
- （可选）`effect_rank`：效果品阶（用于参考）

#### 设计步骤

1. **提取关键信息**  
   - 从条目中获取 `level` 和 `magic`。  
   - 统计“技能/属性”数量：  
     - 技能列表长度记为 `skill_count`。  
     - 若 `attributes` 不为空（即有反应属性），则将其视为一个额外的“技能/属性”项。  
     - 总有效项数 `total_items = skill_count + (1 if attributes不为空 else 0)`。

2. **确定配置类型**  
   - 根据 `total_items` 选择对应配置列：  
     - `total_items = 1` → 使用 **1增配置** 列。  
     - `total_items = 2` → 使用 **2增配置** 列。  
     - `total_items ≥ 3` → 使用 **多增配置** 列。

3. **查表获取数值总数和上限**  
   - 在表格中找到与 `level` 和 `magic` 匹配的行：  
     - 表格按等级分组，每个等级内有多行对应不同的 `magic` 值。  
     - 找到 `magic` 值等于输入 `magic` 的行（注意表格中magic值可能为0.5、1.5等，精确匹配）。  
   - 从该行读取：  
     - `数值总数`（即攻击+防御的总和）  
     - `攻击上限`（如果存在，如“(-1) 9”表示上限为9；若为“-”则无上限限制）  
     - `防御上限`（同理）  
   - 同时可记录该行对应的 `效果品阶` 作为参考。

4. **分配攻击和防御**  
   - 总和必须等于 `数值总数`。  
   - 攻击力 ≤ `攻击上限`（若有），防御力 ≤ `防御上限`（若有）。  
   - 根据怪物定位和技能特点进行合理分配：  
     - 若技能偏向防御（如增加防御、护盾），可适当提高基础防御。  
     - 若技能偏向进攻或需要积攒灵力（如依赖攻击触发效果），可提高攻击。  
     - 也可考虑平衡型分配，但总和必须固定。  
   - 示例：数值总数8，攻击上限6，防御上限7，可分配为攻击2防御6，或攻击1防御7等。

5. **验证**  
   - 确保攻击、防御均为非负整数，且符合上限和总和要求。

#### 注意事项
- 表格中 `magic` 值精确到小数点后一位，需严格匹配。  
- 若 `magic` 值未在表格中列出（通常表格已覆盖常见值），则取最接近的较小值（向下兼容），但需确保规则一致性，建议先与设计者确认。  
- 反应属性视为一个独立的“技能/属性”项，与主动技能并列计算配置类型。  
- 分配时考虑怪物实战强度，避免极端分配（如0/8纯防御可能导致无法输出，但若技能依赖防御也可接受）。  
- 最终数值将填入怪物条目的 `attack` 和 `defence` 字段。

#### 示例回顾
**输入条目**：
```
20260308-00447-ma3d47f0|某|0|雷||0|0|0.5|{}|[{"name":"某","energy_cost":0,"effect":"破坏我方场上1张卡牌，对方信仰-7"},{"name":"某","energy_cost":1,"effect":"防御力+7直到对方回合结束"}]|...|
```
- level=0, magic=0.5, 技能2个，无反应属性 → total_items=2 → 2增配置。
- 查表0级 magic=0.5 行（2增配置列对应数值总数8，攻击上限6，防御上限7）。
- 分配：考虑到有一个技能是临时+7防御，基础防御可设较高，故攻击1防御7（总和8，均未超限）。

**输出**：attack=1, defence=7。
